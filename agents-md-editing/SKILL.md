# Editing AGENTS.md in Flockctl

AGENTS.md is **plain human-owned markdown**. Flockctl never rewrites it. At session start the daemon reads up to three layers, merges them in memory, and injects the result as the agent's system-prompt prefix — the files on disk stay byte-for-byte what humans committed.

## No AUTO-GENERATED headers

Anything you see (or remember seeing) like:

```
<!-- AUTO-GENERATED by Swarmctl from .flockctl/AGENTS.md — do not edit. -->
<!-- Edit .flockctl/AGENTS.md (project source) instead; this file is regenerated. -->
```

is **stale legacy crust** from the retired reconciler. The current model has no reconciler, no source-vs-generated split, no `.flockctl/AGENTS.md` private layer. If you find such a header in a tracked file, just delete it — nothing in Flockctl will rewrite it. The merge for the system prompt happens in `agent-guidance-loader` at runtime; it does **not** mutate files on disk.

## Layers (merged in order)

1. user — `~/flockctl/AGENTS.md`
2. workspace-public — `<ws>/AGENTS.md`
3. project-public — `<proj>/AGENTS.md`

Workspace chats see only 1–2. Missing / empty / directory files are skipped.

## Which layer?

- My-machine preference → **1 user**.
- Team rule across every project in the workspace → **2 workspace-public** (commit).
- Rule specific to a single project → **3 project-public** (commit).

Narrower wins; append semantics — later layers can refine or contradict earlier ones in prose.

## CLAUDE.md

Optional alias: a sibling `CLAUDE.md` symlink → `AGENTS.md` is fine for non-Flockctl agents (Codex, Cursor, plain Claude Code) that look for `CLAUDE.md`. Flockctl itself only reads `AGENTS.md`. Don't make `CLAUDE.md` a separate file — drift is a guarantee.

## API (`:scope` = `projects` | `workspaces`)

- `GET /:scope/:id/agents-md` → `{ layers: { "<scope>-public": {present,bytes,content} } }`.
- `PUT /:scope/:id/agents-md` — `{ content }`. Empty body deletes the file. >256 KiB → `413`. **No cascade** — workspace edits never touch project files.
- `GET /:scope/:id/agents-md/effective` → `{ layers[], totalBytes, truncatedLayers, mergedWithHeaders }` — the in-memory merged view, only used for inspection / system-prompt injection.

## CLI

- `flockctl agents migrate <path>` — one-shot migration from the legacy reconciler layout and/or retired `.flockctl/AGENTS.md` private layer. Strips AUTO-GENERATED headers and folds private content into the public file.
- `flockctl agents show <path> [--workspace] [--effective|--layers]` — inspect merged guidance.

## Caps

Per-layer PUT **256 KiB** (`413` above). Total merged **1 MiB**; overflowing layers named in `truncatedLayers[]`.

## Legacy hotspots in code

The runtime layering loader (`src/services/agent-session/agent-guidance-loader.ts`) reads only `<root>/AGENTS.md` — never `.flockctl/AGENTS.md`. A few spots still mention the retired private path; if you touch them, fix them to write/refer to the public root `AGENTS.md` instead:

- `src/services/project-import.ts` — `applyAdoptAgentsMd` and `applyMergeClaudeMd` currently write to `<project>/.flockctl/AGENTS.md`. That destination is no longer loaded by the daemon, so adopted/merged content silently disappears at session start. New code should target `<project>/AGENTS.md` (and any merge tests must follow).
- `src/cli-commands/project.ts` — `--adopt-agents-md` / `--merge-claude-md` flag help text still says `.flockctl/AGENTS.md`.
- `ui/src/pages/projects-components/NewProjectDialog.tsx` — labels `"Move AGENTS.md → .flockctl/AGENTS.md"` / `"Merge CLAUDE.md into .flockctl/AGENTS.md"`.
- `docs/CLI.md` — flag table reflects the legacy destination.
- `tests/cli-docker/{test-agents,test-project-add}.ts` and `tests/cli-docker/fixtures/repo-with-claude-md-same/` — these are legacy-by-design (they exercise `flockctl agents migrate` and AUTO-GENERATED-header detection). **Do not "modernize" them** — they're the regression tests that prove migration still works for old projects.
